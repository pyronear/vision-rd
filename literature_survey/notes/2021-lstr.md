# Long Short-Term Transformer for Online Action Detection

**Authors:** Xu et al.
**Year/Venue:** 2021, NeurIPS
**PDF:** `pdfs/2021-LSTR-Long-Short-Term-Transformer-Online-Action-Detection-Xu-et-al.pdf`

## Key Idea

A dual-memory Transformer for online action detection that maintains separate long-term (up to 1024s) and short-term (8-32s) memories. The encoder compresses long-term history into a fixed-size representation, and the decoder attends to both memories for classification. Uses sinusoidal positional encodings based on relative time, naturally handling variable frame spacing.

## Temporal Modeling

**Sinusoidal positional encoding** based on relative time offset:
- Each frame at position τ receives embedding s_τ based on `τ = T - t` (time relative to current)
- This is frame-rate agnostic -- handles variable spacing
- Position encodes "how long ago" not "which frame index"

**Memory management:**
- Frames enter short-term memory (FIFO, mS slots)
- When short-term is full, oldest frame graduates to long-term
- Long-term is also FIFO (mL slots), oldest gets dropped
- Performance plateaus at mL = 256-512 seconds

**Cross-attention:** Uses decomposed attention (A' and A") to reduce compute.

## Results

| Dataset | mAP/metric |
|---------|-----------|
| THUMOS'14 | 69.5 mAP |
| TVSeries | 86.1 cAP |
| HDD | - |

**Runtime:** 91.6 FPS on V100 (total system including feature extraction)

**Memory length ablation:** performance increases up to mL=256-512, then plateaus. 8 minutes of long-term context is sufficient for action detection.

## Applicability to Pyronear

**High relevance -- LSTR is the conceptual predecessor to TeSTra.**

**What fits:**
1. **Relative positional encoding** handles 30s frame intervals natively. The model doesn't assume fixed frame rate -- it encodes "how long ago was this frame?" via sinusoidal embeddings.
2. **Dual memory maps to smoke detection:**
   - Short-term (last 2-4 minutes): "is smoke appearing/growing right now?"
   - Long-term (last 30-60 minutes): "what did this area look like before? is this a new event?"
3. **Online/streaming design:** processes one frame at a time as it arrives every 30s.
4. **512s long-term memory** = ~17 frames at 30s intervals. This covers the critical smoke evolution window.

**LSTR vs TeSTra for Pyronear:**
- LSTR has O(T) cost for long-term attention → grows with history length
- TeSTra fixes this with O(1) exponential smoothing
- At 30s intervals, T is small (120 frames/hour), so LSTR's O(T) is manageable
- LSTR is simpler to implement and may be sufficient for Pyronear's scale

**Limitation:** LSTR uses two-stream input (RGB + optical flow). The optical flow stream breaks at 30s. Replace with background subtraction or frame difference channel.

## Takeaways for Implementation

1. **Dual memory is a powerful design pattern** for smoke detection. Consider implementing even a simple version: keep last 4 frames as short-term, maintain an exponential moving average of features as long-term.
2. **Sinusoidal positional encoding by relative time** is the right choice for irregular/sparse frame intervals.
3. **91.6 FPS** means the temporal model adds negligible latency. On a server GPU processing one frame every 30s, LSTR is vastly over-powered.
4. **LSTR is simpler than TeSTra** -- start here, upgrade to TeSTra if you need truly unbounded history.
