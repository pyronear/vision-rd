# YOLOv10: Real-Time End-to-End Object Detection

- **Authors:** Ao Wang, Hui Chen, Lihao Liu, Kai Chen, Zijia Lin, Jungong Han, Guiguang Ding
- **Year/Venue:** 2024, NeurIPS 2024
- **PDF:** `/mnt/data/ssd_1/earthtoolsmaker/projects/pyronear/papers/pdfs/2024-YOLOv10-Real-Time-End-to-End-Object-Detection-Wang-et-al.pdf`

## Key Idea

YOLOv10 advances the YOLO family by eliminating the need for Non-Maximum Suppression (NMS) during inference through a consistent dual label assignments strategy, and by introducing a holistic efficiency-accuracy driven model design. The NMS-free approach uses dual heads during training (one-to-many for rich supervision, one-to-one for inference) with a consistent matching metric that aligns both heads. Architectural innovations include lightweight classification heads, spatial-channel decoupled downsampling, rank-guided block design, large-kernel convolutions, and partial self-attention (PSA).

## Architecture / Content

```
YOLOv10 Architecture:
+--------+     +------+     +------------------+
| Input  | --> | Stem | --> | Backbone (stages) |
+--------+     +------+     +------------------+
                                    |
                              +-----+-----+
                              |    PAN     |  (Path Aggregation Network)
                              +-----------+
                                    |
                    +---------------+---------------+
                    |                               |
             One-to-Many Head              One-to-One Head
             (training only)               (inference only)
             [Regression]                  [Regression]
             [Classification]             [Classification]
                    |                               |
              NMS required                    NMS-FREE
              (discarded at                  (used at
               inference)                    inference)

Key efficiency innovations:
  1. Lightweight cls head: depthwise-sep conv (3x3 + 1x1)
     -> cls head cost: 5.95G -> much less
  2. Spatial-channel decoupled downsampling:
     PW conv (channel) -> DW conv (spatial)
     instead of single strided 3x3 conv
  3. Rank-guided block design:
     CIB (Compact Inverted Block) replaces bottleneck
     in low-rank (redundant) stages
  4. Large-kernel conv: 7x7 DW in CIB (deep stages only)
  5. PSA (Partial Self-Attention): only on Stage 4
     (lowest resolution), channels split in half

Model scales: N / S / M / B / L / X
```

### Key improvements over YOLOv8:
- **N scale:** 28% fewer params, 23% less FLOPs, 1.2% better AP, 70% lower latency
- **S scale:** 36% fewer params, 24% less FLOPs, 1.4% better AP, 65% lower latency
- **M scale:** 41% fewer params, 25% less FLOPs, 0.5% better AP, 50% lower latency

### Ablation (YOLOv10-S):
- NMS-free training reduces end-to-end latency from 7.07ms to 2.49ms
- Efficiency-driven design: -11.8M params, -20.8 GFLOPs, -0.65ms latency
- Accuracy-driven design: +1.8 AP with only +0.18ms latency

## Applicability to Pyronear

**Highly applicable -- direct upgrade path from current YOLOv8.**

- **NMS-free inference on RaspPi:** Eliminating NMS post-processing is a significant win for edge deployment. NMS adds latency and complexity; YOLOv10's one-to-one head produces direct predictions without it.
- **YOLOv10-N as edge model:** With only 2.3M params and 6.7 GFLOPs (vs. YOLOv8-N's 3.2M/8.7G), YOLOv10-N is 28% smaller and 70% faster while being 1.2% more accurate. This is ideal for the RaspPi's limited compute.
- **YOLOv10-S for better accuracy at still-fast speed:** 7.2M params, 46.3% AP, 2.49ms latency on T4 GPU. Could be viable on RaspPi depending on acceptable frame processing time (30s between frames gives ample budget).
- **Consistent dual assignments training:** The training technique is model-agnostic and can be applied to train any YOLO variant. Could improve the existing YOLOv8 training without changing the model.
- **Spatial-channel decoupled downsampling:** Preserves more information during downsampling, which is critical for detecting small smoke plumes that occupy ~2% of image area.

## Takeaways for Implementation

1. **Evaluate YOLOv10-N as a drop-in replacement for YOLOv8-N/S on the RaspPi.** Expected: same or better detection quality with faster inference and lower memory. The NMS-free design simplifies the inference pipeline.
2. **YOLOv10-S on server side** for the second-stage verification could replace or supplement the ResNet+LSTM pipeline, offering better feature extraction for small objects.
3. **The consistent dual assignment training strategy** can be applied even if staying with YOLOv8 architecture -- it is a training-time technique that improves the one-to-one head.
4. **Partial Self-Attention (PSA)** is only applied at the lowest resolution stage (Stage 4), adding global context with minimal overhead. This could help with the amorphous, context-dependent nature of smoke.
5. **Large-kernel convolutions** (7x7 DW) in deep stages enlarge the receptive field, improving detection of diffuse objects like smoke without heavy compute cost.
6. **Model available at:** https://github.com/THU-MIG/yolov10 -- ready for fine-tuning on PyroNear2025 dataset.
7. **Consider YOLOv10-B** (the -B variant between M and L) for best edge-vs-accuracy tradeoff if the RaspPi can handle ~19.4M params.
