# MATR: Online Temporal Action Localization with Memory-Augmented Transformer

- **Authors:** Youngkil Song, Dongkeun Kim, Minsu Cho, Suha Kwak
- **Year/Venue:** 2024 / ECCV (arXiv:2408.02957)
- **PDF:** `../pdfs/2024-MATR-Memory-Augmented-Transformer-Online-Action-Localization-Song-et-al.pdf`

## Key Idea

MATR addresses online temporal action localization (On-TAL) -- detecting action start/end times in streaming video using only past and current frames (no future lookahead). Its core innovation is a memory queue that selectively stores past segment features via a learned FLAG token, enabling long-term context without increasing the input window size. It uses two separate Transformer decoders: an end decoder to detect action endings in the current segment, and a start decoder that scans the memory queue to locate the corresponding action start. Separate class and boundary queries decouple classification from localization.

## Results

- SOTA on both THUMOS14 and MUSES for online TAL
- Comparable to some offline TAL methods (no NMS needed)
- Memory queue + FLAG-based selective storage outperforms fixed-window approaches
- Decoupled class/boundary queries improve both classification and localization
- Robust to input segment size hyperparameter (unlike prior On-TAL methods)

## Applicability to Pyronear

**What transfers:**
- **Online streaming paradigm matches Pyronear exactly.** Pyronear processes frames as they arrive (every 30s) and must decide in real-time without future information -- this is precisely the On-TAL setting.
- **Memory queue concept is directly applicable.** Store features from past frames/segments, selectively keeping those flagged as "potentially containing smoke." When a new detection arrives, scan the memory to determine when the smoke event started.
- **FLAG token for selective storage** is elegant for Pyronear: only store segments where YOLOv8 detected something interesting, avoiding memory waste on clear-sky frames.
- **Decoupled queries** (class vs boundary) map to Pyronear's needs: "is this smoke?" (classification) vs "when did it start?" (temporal localization) are separate concerns.
- **No NMS required** simplifies the inference pipeline.

**What doesn't transfer:**
- Designed for action localization with dense frame features (extracted from a video backbone). Pyronear has very sparse frames (30s apart).
- The temporal granularity of action boundaries (frame-level precision) may not be needed -- Pyronear mainly needs to confirm/reject and roughly timestamp.
- Requires a pre-trained feature extractor backbone; the paper uses existing video features, not raw frames.
- Training requires temporal action datasets with start/end annotations.

## Takeaways for Implementation

1. **Implement a memory queue on the server.** Store per-frame features from YOLOv8 detections in a FIFO buffer. When a new suspicious frame arrives, cross-attend to the memory to check temporal consistency (smoke growing over 2-4 frames = real fire).
2. **FLAG-based gating:** Train a binary classifier on each frame's features to decide whether to store in memory. This keeps the buffer lean -- only frames with potential smoke are retained.
3. **Two-decoder approach for Pyronear:** One decoder confirms "is this a real smoke event?" (end detection), and another scans memory to find "when did it start?" for alert timestamping.
4. **Segment size flexibility:** MATR is robust to segment size, which is useful for Pyronear where the effective "segment" is variable (could be 1 frame or a batch of frames).
5. **This is the most architecturally relevant paper for Pyronear's temporal verification pipeline.** The memory-augmented online processing paradigm directly maps to the problem of confirming YOLOv8 detections over time.
